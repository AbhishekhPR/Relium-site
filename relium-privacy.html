Privacy model

Designed to work without customer records.

Relium is metadata-first by design. The agent can collect aggregate metrics and sanitized SQL evidence close to the customer environment instead of moving raw rows into an external system.

That means teams can investigate reliability issues using counts, schema changes, null rates, duplicate counts, SQL structure, and lineage rather than customer-level records or query result exports.

Relium uses row counts, null rates, duplicate counts, schema metadata, SQL structure, and lineage. It does not require raw customer records or query result exports.
